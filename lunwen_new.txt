水蜜桃垂直电商平台的设计与实现

摘要

随着生鲜电商市场的快速发展，水蜜桃作为典型的"高敏、易损、非标"农产品，其电商化面临着独特的挑战。本文设计并实现了一个基于Spring Boot和Vue.js的水蜜桃垂直电商平台，采用前后端分离架构，实现了用户管理、商品管理、购物车、订单管理、地址管理等核心功能。

系统采用Spring Boot 3.2.2作为后端框架，Vue 3 + TypeScript + Vite作为前端技术栈，PostgreSQL作为数据库。已实现的功能包括：用户注册与登录、商品浏览与管理、购物车操作、订单创建与管理、收货地址管理、模拟支付流程等。此外，本课题还完成了协同过滤推荐算法的代码开发，构造100个模拟用户，依据水蜜桃的产地、品种、商品热度生成用户行为数据，完成算法流程的模拟验证；完成了基于Selenium的爬虫代码编写，可从惠农网获取水蜜桃商品数据。需要说明的是，由于本科毕业设计的时间与资源限制，支付功能采用模拟方式实现（真实支付接口需要企业资质认证），推荐系统为模拟验证系统，爬虫系统暂未实现定时自动入库，本版本仅实现消费者端，商户端与管理员端暂未实现。

本文详细阐述了系统的需求分析、架构设计、数据库设计、核心功能实现和功能测试。测试结果表明，系统已实现的核心功能运行正常，能够满足水蜜桃垂直电商的基本业务需求。本文的研究为生鲜垂直电商提供了可参考的技术方案和实践经验。

关键词：水蜜桃电商；垂直电商；Spring Boot；Vue.js；推荐系统；协同过滤

Abstract

With the rapid development of the fresh food e-commerce market, peaches, as typical "highly sensitive, perishable, and non-standard" agricultural products, face unique challenges in e-commerce. This paper designs and implements a peach vertical e-commerce platform based on Spring Boot and Vue.js, adopting a front-end and back-end separation architecture, implementing core functions such as user management, product management, shopping cart, order management, and address management.

The system uses Spring Boot 3.2.2 as the back-end framework, Vue 3 + TypeScript + Vite as the front-end technology stack, and PostgreSQL as the database. The implemented functions include: user registration and login, product browsing and management, shopping cart operations, order creation and management, shipping address management, and simulated payment process. In addition, this project has completed the code development of collaborative filtering recommendation algorithm, constructing 100 simulated users, generating user behavior data based on peach origin, variety, and product popularity, and completing the simulation verification of the algorithm process; completed the crawler code based on Selenium, which can obtain peach product data from Huinong.com. It should be noted that due to the time and resource constraints of undergraduate graduation design, the payment function is implemented in a simulated manner (real payment interfaces require enterprise qualification certification), the recommendation system is a simulation verification system, the crawler system has not yet implemented scheduled automatic data storage, and this version only implements the consumer end, while the merchant end and administrator end are not yet implemented.

This paper elaborates on the system's requirements analysis, architecture design, database design, core function implementation, and functional testing. Test results show that the implemented core functions operate normally and can meet the basic business needs of peach vertical e-commerce. The research in this paper provides reference technical solutions and practical experience for fresh food vertical e-commerce.

Keywords: Peach E-commerce; Vertical E-commerce; Spring Boot; Vue.js; Recommendation System; Collaborative Filtering


第1章 绪论

1.1 研究背景与动机

随着我国农业现代化进程的加速和居民消费升级，生鲜电商已成为农产品流通的重要渠道。根据国家统计局数据，2025年全国生鲜电商市场规模已突破1.2万亿元，同比增长28.7%。然而，在众多生鲜品类中，水蜜桃作为典型的"高敏、易损、非标"农产品，其电商化面临着远超普通水果的独特挑战。

水蜜桃具有以下显著特性：
1. 季节性强：主要集中在6-9月上市，销售周期短
2. 易腐性高：常温下保鲜期仅3-5天，对冷链物流要求极高
3. 非标准化：品种繁多（超过200种），不同产区的甜度、口感、成熟期差异显著
4. 消费者要求高：对"产地直采、新鲜度、溯源透明"有较高要求

传统综合电商平台（如京东、天猫生鲜频道）采用"中心仓+城市分仓"的通用模式，难以适应水蜜桃"产地直发、冷链零中断"的需求。从果园采摘到消费者餐桌，通常需经过多级经销商、中心仓中转、城市分拣等环节，导致平均流通时间超过48小时，新鲜度损耗率较高。同时，消费者难以实时获取产地信息、采摘日期等关键数据，信任危机频发，退货率居高不下。

本研究的动机正是针对上述水蜜桃特有痛点，构建一个垂直化、智能化的电商平台——peach-shop。该平台以"产地直采+个性化推荐"为核心设计理念，通过技术手段实现从果园到餐桌的"信息透明"与"极速流转"，为水蜜桃产业提供可参考的数字化解决方案。

1.2 国内外研究现状

1.2.1 国外垂直电商发展现状

国外生鲜电商起步较早，以美国Blue Apron、HelloFresh为代表的食材订阅垂直平台，以及日本的Oisix生鲜平台，强调"产地直采+冷链宅配"。这些平台通过IoT传感器实现冷链温湿度实时监控，供应链环节压缩至2-3级，损耗率控制在5%以内。但其主要针对标准化、高单价食材（如有机蔬菜、海鲜），对水蜜桃这类季节性极强、非标特征突出的水果覆盖较少。

1.2.2 国内垂直电商发展现状

国内综合电商平台（京东、天猫）凭借强大的物流网络（京东冷链、菜鸟物流）占据生鲜市场主导地位，但采用"中心仓模式"，对水蜜桃等时令水果的适应性有待提升。垂直品类平台则展现出一定优势：

1. 百果园：聚焦高端水果，通过"果园直采+自营冷链+门店前置仓"模式，将水蜜桃流通时间压缩至24小时以内
2. 叮咚买菜：通过预售+产地直发降低库存压力
3. 盒马鲜生：采用"店仓一体"模式，实现快速配送

然而，现存垂直平台仍存在一些共性短板：
1. 推荐系统对水蜜桃多维特征（品种、甜度、产地）捕捉不足
2. 多角色协同机制有待完善
3. 溯源功能多停留在基础层面，缺乏与物流、支付、营销的深度融合

1.3 研究目标与内容

本研究的核心目标是：设计并实现一个以水蜜桃为垂直品类的电商平台，通过技术手段解决"信息透明"与"极速流转"两大核心问题。

具体研究内容包括：
1. 需求分析与架构设计：梳理消费者端的功能需求，设计前后端分离架构
2. 核心功能实现：实现用户管理、商品管理、购物车、订单管理、地址管理等核心功能
3. 个性化推荐系统：完成基于用户的协同过滤推荐算法代码开发，构造模拟用户与行为数据，验证算法流程的可行性
4. 数据采集系统：完成基于Selenium的爬虫代码编写，实现从惠农网获取水蜜桃商品数据的功能
5. 系统测试：完成核心功能的功能测试，验证系统基本功能的正确性

需要说明的是，由于本科毕业设计的时间与资源限制，本研究在以下方面存在局限性：
1. 支付功能采用模拟方式实现，未接入真实支付接口（真实支付接口需要企业资质认证）
2. 物流功能仅完成订单状态设计，未接入物流API
3. 溯源功能仅完成数据库字段设计与流程规划，未实现二维码生成与查询页面
4. 推荐系统为模拟验证系统，使用人工构造的模拟用户行为数据
5. 爬虫系统暂未实现定时自动入库
6. 本版本仅实现消费者端，商户端与管理员端暂未实现

第2章 相关技术介绍

2.1 后端技术栈

2.1.1 Spring Boot框架

Spring Boot是由Pivotal团队提供的框架，其设计目的是用来简化新Spring应用的初始搭建以及开发过程。该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置。

Spring Boot的核心特性包括：
1. 独立运行的Spring项目：可以以jar包形式独立运行，内嵌Tomcat、Jetty等Servlet容器
2. 自动配置：Spring Boot会根据项目依赖自动配置Spring应用
3. 提供生产级的监控：如指标、健康检查和外部化配置
4. 无代码生成和XML配置：不需要XML配置文件，采用注解方式

本项目采用Spring Boot 3.2.2版本，基于Spring Framework 6和Jakarta EE 10，使用"约定优于配置"理念，通过Starter机制快速集成Web、Data JPA等组件。

2.1.2 Spring Data JPA

Spring Data JPA是Spring Data家族的一部分，可以轻松实现基于JPA的存储库。该模块处理对基于JPA的数据访问层的增强支持，使构建使用数据访问技术的Spring应用程序变得更加容易。

Spring Data JPA的主要特性：
1. 支持基于XML和基于注解的实体映射
2. 支持Example查询、Specification查询
3. 支持分页和排序
4. 支持原生SQL查询

本项目使用Spring Data JPA实现对PostgreSQL数据库的访问，通过Repository接口定义数据访问方法，简化了数据访问层的开发。

2.1.3 PostgreSQL数据库

PostgreSQL是一个功能强大的开源对象关系数据库系统，具有超过35年的活跃开发历史，以其可靠性、功能健壮性和性能而享有盛誉。

PostgreSQL的主要优势：
1. 支持复杂查询：支持SQL标准的大部分特性，包括复杂查询、外键、触发器、视图等
2. 扩展性强：支持自定义数据类型、自定义函数、自定义操作符等
3. 数据完整性：支持事务、MVCC（多版本并发控制）、行级锁等
4. 开源免费：PostgreSQL许可证允许免费使用、修改和分发

本项目选用PostgreSQL作为主数据库，主要考虑其在处理复杂查询和数据完整性方面的优势，适合水蜜桃电商平台的多维度数据管理需求。

Vue.js是一套用于构建用户界面的渐进式JavaScript框架。与其他大型框架不同的是，Vue被设计为可以自底向上逐层应用。Vue的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。

Vue.js的核心特性：
1. 响应式数据绑定：Vue.js的核心是一个响应式系统，当数据发生变化时，视图会自动更新
2. 组件化：Vue.js允许将页面拆分成多个可重用的组件，每个组件都有自己的逻辑和样式
3. 虚拟DOM：Vue.js使用虚拟DOM来提高渲染性能
4. 指令系统：Vue.js提供了丰富的指令（如v-if、v-for、v-bind等），简化了DOM操作

2. 更好的IDE支持：提供代码补全、重构等功能
3. 更好的代码组织：支持接口、泛型、枚举等特性
4. 兼容JavaScript：可以与JavaScript代码无缝集成

本项目前端代码全部采用TypeScript编写，提高了代码的可维护性和可读性。

2.3 推荐算法

2.3.1 协同过滤算法

协同过滤（Collaborative Filtering）是推荐系统中应用较广泛的算法之一，其核心思想是利用用户的历史行为数据来发现用户的兴趣偏好，进而推荐用户可能感兴趣的物品。

协同过滤算法主要分为两类：
1. 基于用户的协同过滤（User-based CF）：找到与目标用户兴趣相似的其他用户，推荐这些用户喜欢的物品
2. 基于物品的协同过滤（Item-based CF）：找到与目标用户喜欢的物品相似的其他物品，推荐这些物品

本项目采用基于用户的协同过滤算法，通过计算用户之间的相似度，为目标用户推荐相似用户喜欢的商品。

2.3.2 相似度计算方法

常用的相似度计算方法包括：
1. 余弦相似度：计算两个向量之间的夹角余弦值
2. 皮尔逊相关系数：衡量两个变量之间的线性相关程度
3. Jaccard相似系数：衡量两个集合的相似度

本项目使用余弦相似度计算用户之间的相似度，公式如下：

sim(u, v) = cos(θ) = (r_u · r_v) / (||r_u|| * ||r_v||)

其中，r_u和r_v分别表示用户u和用户v的评分向量。

Selenium是一个用于Web应用程序测试的工具，支持多种浏览器（如Chrome、Firefox、Edge等），可以模拟用户在浏览器中的操作。

Selenium的主要特性：

数据解析是爬虫的核心环节，常用的解析技术包括：

```java
@PostMapping("/add")
public ResponseEntity<Map<String, Object>> addToCart(@RequestBody Map<String, Object> request) {
    Map<String, Object> response = new HashMap<>();
    
    try {
        Long userId = Long.parseLong(request.get("userId").toString());
        String productCode = request.get("productCode").toString();
        Integer quantity = Integer.parseInt(request.get("quantity").toString());
        
        ShoppingCart existingItem = shoppingCartService.findByUserIdAndProductCode(userId, productCode);
        
        if (existingItem != null) {
            existingItem.setProductNum(existingItem.getProductNum() + quantity);
            shoppingCartService.update(existingItem);
        } else {
            ShoppingCart cartItem = new ShoppingCart();
            cartItem.setUserId(userId);
            cartItem.setProductCode(productCode);
            cartItem.setProductNum(quantity);
            cartItem.setIsChecked(1);
            shoppingCartService.save(cartItem);
        }
        
        response.put("code", 0);
        response.put("msg", "添加成功");
        return ResponseEntity.ok(response);
        
    } catch (Exception e) {
        response.put("code", 500);
        response.put("msg", "添加失败: " + e.getMessage());
        return ResponseEntity.status(500).body(response);
    }
}
```

```python
        """加载训练好的模型和数据"""
        with open('models/user_similarity.pkl', 'rb') as f:
            self.user_similarity = pickle.load(f)
        with open('models/user_item_matrix.pkl', 'rb') as f:
            self.user_item_matrix = pickle.load(f)
        self.products = pd.read_csv('data/products.csv')
    
    def get_user_recommendations(self, user_id, top_n=10):
        """为指定用户生成推荐"""
        if user_id not in self.user_item_matrix.index:
            return self.get_popular_products(top_n)
        
        user_similarities = self.user_similarity[user_id]
        similar_users = user_similarities.argsort()[::-1][1:top_n+1]
        
        user_purchases = self.user_item_matrix.loc[user_id]
        purchased_items = user_purchases[user_purchases > 0].index
        
        recommendations = {}
        for similar_user in similar_users:
            similar_user_purchases = self.user_item_matrix.loc[similar_user]
            for item in similar_user_purchases.index:
                if item not in purchased_items and similar_user_purchases[item] > 0:
                    if item not in recommendations:
                        recommendations[item] = 0
                    recommendations[item] += user_similarities[similar_user] * similar_user_purchases[item]
        
        sorted_recommendations = sorted(recommendations.items(), key=lambda x: x[1], reverse=True)
        return [item[0] for item in sorted_recommendations[:top_n]]
    
    def get_popular_products(self, top_n=10):
        """获取热门商品"""
        item_popularity = self.user_item_matrix.sum(axis=0)
        popular_items = item_popularity.sort_values(ascending=False).head(top_n).index
        return list(popular_items)
```

5.3.3 模拟验证系统说明

本推荐系统为模拟验证系统，用于验证协同过滤算法流程的可行性。系统构造100个模拟用户，依据水蜜桃的产地、品种、商品热度，为每位用户生成浏览、加购、收藏、购买四类行为数据，并赋予不同权重，形成用户-商品行为矩阵。

模拟数据设计说明：
1. 用户数量：100个模拟用户
2. 商品数量：20个水蜜桃商品（对应不同产地和品种）
3. 行为类型：浏览、加购、收藏、购买
4. 行为权重：浏览(1.0)、加购(2.0)、收藏(2.5)、购买(3.0)
5. 数据构造依据：
   - 产地偏好：将用户分为5组，每组对应不同产地的偏好（阳山水蜜桃、蒙阴蜜桃、深州蜜桃、奉化水蜜桃、炎陵黄桃）
   - 品种偏好：在产地基础上，根据品种热度分配行为概率
   - 商品热度：热门商品获得更多行为数据的概率更高

模型训练代码：

```python
import pandas as pd
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity
import pickle

def train_model():
    """训练推荐模型"""
    interactions = pd.read_csv('data/interactions.csv')
    products = pd.read_csv('data/products.csv')
    users = pd.read_csv('data/users.csv')
    
    user_item_matrix = interactions.pivot_table(
        index='user_id',
        columns='product_id',
        values='rating',
        fill_value=0
    )
    
    user_similarity = cosine_similarity(user_item_matrix)
    user_similarity = pd.DataFrame(
        user_similarity,
        index=user_item_matrix.index,
        columns=user_item_matrix.index
    )
    
    item_similarity = cosine_similarity(user_item_matrix.T)
    item_similarity = pd.DataFrame(
        item_similarity,
        index=user_item_matrix.columns,
        columns=user_item_matrix.columns
    )
    
    with open('models/user_similarity.pkl', 'wb') as f:
        pickle.dump(user_similarity, f)
    with open('models/item_similarity.pkl', 'wb') as f:
        pickle.dump(item_similarity, f)
    with open('models/user_item_matrix.pkl', 'wb') as f:
        pickle.dump(user_item_matrix, f)
    
    print("模型训练完成！")

if __name__ == '__main__':
    train_model()
```

基于模拟用户行为数据，模型训练生成了以下结果：

2. 物品相似度矩阵：20×20的对称矩阵，记录商品之间的关联度

使用Selenium实现水蜜桃商品数据爬取：

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
import pandas as pd
import time

class PeachCrawler:
    def __init__(self):
        self.driver = webdriver.Chrome()
        self.wait = WebDriverWait(self.driver, 10)
        
    def crawl_products(self, url):
        """爬取商品数据"""
        self.driver.get(url)
        time.sleep(2)
        
        products = []
        product_elements = self.driver.find_elements(By.CLASS_NAME, 'product-item')
        
        for element in product_elements:
            try:
                product = {
                    'name': element.find_element(By.CLASS_NAME, 'product-name').text,
                    'price': element.find_element(By.CLASS_NAME, 'product-price').text,
                    'image': element.find_element(By.TAG_NAME, 'img').get_attribute('src'),
                    'description': element.find_element(By.CLASS_NAME, 'product-desc').text
                }
                products.append(product)
            except Exception as e:
                print(f"爬取商品失败: {e}")
                continue
        
        return products
    
    def save_to_csv(self, products, filename):
        """保存数据到CSV文件"""
        df = pd.DataFrame(products)
        df.to_csv(filename, index=False, encoding='utf-8-sig')
        print(f"数据已保存到 {filename}")
    
    def close(self):
        """关闭浏览器"""
        self.driver.quit()

if __name__ == '__main__':
    crawler = PeachCrawler()
    try:
        products = crawler.crawl_products('https://www.huinong.com/peach-products')
        crawler.save_to_csv(products, 'data/products.csv')
    finally:
        crawler.close()
```

附录B 系统使用说明

B.1 用户注册与登录

1. 访问系统首页，点击"注册"按钮
2. 填写用户名、密码等信息
3. 点击"注册"按钮完成注册
4. 使用注册的用户名和密码登录系统

B.2 商品浏览与购买

1. 在首页查看商品列表
2. 点击商品查看详情（包括品种、产地、甜度、采摘日期等垂直特色属性）
3. 点击"加入购物车"按钮
4. 进入购物车页面，选择商品
5. 点击"去结算"按钮
6. 填写收货地址，提交订单
7. 完成模拟支付

B.3 订单管理

1. 点击"我的订单"查看订单列表
2. 点击订单查看详情
3. 可以取消未支付订单
4. 可以查看订单状态

B.4 个人中心

1. 点击"个人中心"进入个人中心页面
2. 可以查看个人信息
3. 可以管理收货地址
4. 可以查看积分和钱包余额
